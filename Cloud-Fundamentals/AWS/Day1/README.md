##  What is Cloud Computing

Cloud computing is a way of using computing resources like servers, storage, and applications over the internet instead of managing physical hardware.

We can:
- Launch servers  
- Store data  
- Run applications  
- Build and deploy websites  

 It follows a **pay-as-you-use model**, which reduces cost.

---

##  Evolution of Cloud Computing

### Before Cloud
- Companies used physical data centers  
- Required high investment  
- Setup and maintenance were slow  

### Virtualization
- One physical server runs multiple virtual machines  
- Better resource usage  

### Cloud Era
- Started with AWS in 2006  
- Resources can be created instantly  
- No need to manage hardware  

---

##  Traditional IT vs Cloud

| Traditional IT | Cloud |
|---------------|------|
| Physical servers required | No hardware needed |
| High upfront cost | Pay as you use |
| Slow setup | Quick setup |
| Difficult to scale | Easy to scale |
| Manual maintenance | Managed by provider |

---

##  Business Problems Solved by Cloud

Cloud helps organizations to:
- Reduce infrastructure cost  
- Scale during high traffic  
- Expand globally  
- Recover from failures  
- Deploy applications faster  

 Example: E-commerce sites increase servers during big sales.

---

##  Deployment Models

### Public Cloud
- Services available over internet  
- Used by multiple users  

### Private Cloud
- Dedicated to one organization  
- More control and security  

### Hybrid Cloud
- Combination of public and private cloud  

---

##  Service Models

### IaaS (Infrastructure as a Service)
- Provides virtual machines, storage, networking  
- User manages OS and applications  

### PaaS (Platform as a Service)
- Provides platform to develop and run applications  
- No need to manage OS  

### SaaS (Software as a Service)
- Ready-to-use applications  
- Example: Gmail, Google Docs  

---

##  Core Cloud Concepts

- **Elasticity** → Automatically increase or decrease resources  
- **Scalability** → Handle growing workload  
- **High Availability** → System stays online  
- **Fault Tolerance** → Works even if a component fails  

---

##  Shared Responsibility Model

### Cloud Provider is responsible for:
- Physical infrastructure  
- Data centers  
- Networking  

### Customer is responsible for:
- Data security  
- User access  
- Application configuration  

---

##  Why AWS

- Large number of services  
- Global infrastructure

##  AWS Global Infrastructure

- **Regions** → Different geographical locations (e.g., Mumbai, US)  
- **Availability Zones (AZs)** → Multiple data centers within a region  

✔ Improves availability  
✔ Supports disaster recovery  
✔ Ensures better performance
